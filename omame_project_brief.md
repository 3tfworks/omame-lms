# 【お豆奏法プロジェクト】システム要件およびインフラ戦略 引き継ぎ書

このドキュメントは、Mikasel開発で得た知見をもとに策定した「えりな先生の動画プラットフォーム（お豆プロジェクト）」のシステムアーキテクチャおよび開発戦略のマスタープランです。
新しいチャットセッションでAI（ラファエル）と開発をスタートする際は、**必ず最初にこのドキュメントを読み込ませてください**。

---

## 🎯 1. プロジェクトの基本方針（MVP戦略）

*   **目的**: FANTS等の外部高額ツールに依存せず、Mikaselのコード資産（Vercel/Supabase）を流用して「最強の自社製・動画コミュニティプラットフォーム」を爆速で構築する。
*   **フェーズ分けによる爆速ローンチ**:
    *   **Phase 1（最優先）**: オプトLP、商品LP、メインHP、**動画コンテンツ会員サイト（決済含む）**の構築。キャッシュ（現金）を生むコア機能だけを先にローンチする。
    *   **Phase 2（おいおい開発）**: 会員制のコミュニケーションの場（掲示板・チャット等）。複雑な機能は後回しにする。
*   **資産の流用**: Mikaselで構築した「Stripe決済（サブスク管理）」「ログイン・権限管理」「LPのUIパーツ」をごっそり流用し、開発スピードを3倍に引き上げる。

## 🏛️ 2. インフラ・アーキテクチャ（プロの二刀流構成）

外部ツールと自社インフラを組み合わせた、最もコストパフォーマンスと安全性が高い構成を採用する。

1.  **フロントエンド（Web表示）**: **Vercel + Next.js**
    *   爆速の表示スピードと、Netflixのようなサクサク動くUIを実現。絶対に落ちない。
2.  **バックエンド（データベース・認証）**: **Supabase**
    *   顧客データ、パスワード、Stripe決済ステータスの安全な管理。「課金切れ」と同時に動画を見れなくする強固なアクセス制御。
3.  **メール・ドメイン管理**: **エックスサーバー（契約済）**
    *   ドメインのDNS設定を「WebはVercelへ」「メールはエックスサーバーへ」と分割する。
    *   会員からのお問い合わせ窓口（`info@...`）の受信箱として利用し、契約を100%有効活用する。
4.  **動画配信・保存**: **Vimeo（または Cloudflare Stream 等）**
    *   重い動画データをSupabaseに置くとサーバー代が爆発するため、プロの動画CDNを利用。
    *   「自社システム以外では絶対に再生できない（真っ黒になる）」ロックをかけ、動画の流出を完全に防ぐ。

## 💰 3. ランニングコスト試算（破壊的コストメリット）

FANTS等（月額数十万＋手数料）と比較し、自社開発によるインフラ維持費は圧倒的に安く、利益率が極大化する。

*   **Vercel (Pro)**: 約3,000円/月
*   **Supabase (Pro)**: 約3,800円/月
*   **Vimeo (Advanced等)**: 約数千円〜10,000円/月（動画容量による）
*   **エックスサーバー**: 契約済（実質追加ゼロ）
*   **Stripe手数料**: 月額0円（決済額の約3.6%のみ）
*   **【合計】**: **月額 約 1.5万円 以下**

## 🚀 4. 次のアクション（開発スタート手順）

AI（ラファエル）との新チャットで開発を開始するためのステップ：

1.  Supabaseのダッシュボードで、お豆奏法用の **「新しいProject」** を作成し、`Project URL` と `anon key` を取得しておく。
2.  Cursorで「お豆プロジェクト」の空フォルダを開く。
3.  新しいチャットを立ち上げ、AIに**「この引き継ぎ書（omame_project_brief.md）を読んで」**と指示する。
4.  AIが `npx create-next-app` で環境を構築し、Mikaselから決済・ログイン機能を移植する。
