# クラシック演奏Shorts専用リサーチ 実装計画

## 目的

クラシック系の「弾いてみた」Shortsについて、競合調査から曲選定、難所選定、タイトル、冒頭テロップ、撮影カット割りまでを自動生成する。

## 実装内容

1. `standard` と `classical_shorts` の調査モードを実行履歴へ保存する。
2. ShortsモードではYouTube Data APIの短尺検索を使い、3分以内の動画だけを分析対象にする。
3. 再生速度、登録者数に対する外れ値、いいね・コメント反応率からShorts向け競合順位を作る。
4. AI企画生成へ以下の構造化項目を追加する。
   - 作曲家・曲名
   - 難所と演奏区間
   - 冒頭テロップ
   - 目標秒数
   - 撮影カット割り
   - 権利確認メモ
5. 管理画面で通常動画／演奏Shortsを切り替えて実行し、企画カードとCSVに専用項目を表示する。
6. 毎週木曜日10:00（日本時間）にShorts調査を自動実行する。

## 本番反映順序

1. `supabase/migrations/20260713030000_youtube_classical_shorts.sql` をSupabase SQL Editorで実行する。
2. コードをGitへコミットし、Vercel Productionへデプロイする。
3. 管理画面で「演奏Shorts」を選び、「演奏Shortsを自動企画」を実行する。
4. 生成された企画の曲、難所、冒頭テロップ、秒数、カット割り、権利確認メモを確認する。

## 自動実行

- 通常動画: 毎週月曜日 10:00 JST
- クラシック演奏Shorts: 毎週木曜日 10:00 JST

無料AI Gatewayクレジットを使い切った場合、自動チャージは行わず、企画生成ステップが失敗として記録される。
