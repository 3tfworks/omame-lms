## 6. 🆕 次セッションの目玉:Magic Link メール文面の抜本改善

### 6.1 背景

本日のログイン問い合わせ4件(松崎・坂本・森田・齋藤さん)のうち、少なくとも3件が **メール内「ログインする」リンク** に起因。共通課題は2つ:

1. **視認性**:「ログインする」が青い下線テキストで、押せるリンクだと気づかれない(特に 50-70代)
2. **ブラウザ跨ぎ**:LINE 内 / メールアプリ内ブラウザで開くと PKCE エラーで失敗

### 6.2 改善候補

| 施策 | 内容 | 懸念 |
|---|---|---|
| **ボタン化** | メールの「ログインする」を枠+背景色のボタン風デザインに。「タップしてください」の補足文も | §7.4(c) の「テンプレは短く」方針との兼ね合い |
| **ブラウザ注意文** | メール本文に「LINE で開いた方は Safari で開いてください」の一文 | メールが長くなる |
| **配置** | Supabase Dashboard → Authentication → Email Templates → Magic Link | HTML 直編集 |

### 6.3 §7.4(d) LINE 自動応答(合流)

問い合わせ窓口の LINE 公式アカウントで、キーワード自動応答を設定:
- キーワード:「ログイン」「入れない」「ログインできない」「メールが届かない」
- 返信文案:
  > ご連絡ありがとうございます。ログイン方法でお困りの場合は、画像つきの案内ページをご用意しております。
  > https://www.omamepiano.com/ja/help/login
  > 上記をご覧いただいても解決しない場合は、このトーク画面にて状況をお知らせください。
